# Python Project Reporting with Makefile

This **Makefile** automates the preparation of artifacts needed for **evaltech reporting** and **SonarQube analysis**.  
It handles the following tasks:

- Checking for required tools (`poetry`, `pytest`, `coverage`, `cyclonedx-py`, `sonar-scanner`)
- Generating a **CycloneDX SBOM (Software Bill of Materials)** in JSON and XML formats directly from `poetry.lock`
- Running unit tests and producing a **coverage report** (for SonarQube)
- Executing the **SonarQube scanner** to analyze the project

---
## Prequisites

Before running the Makefile, make sure you have the following installed:

1. **Python 3.9+**  
2. **Poetry** (see [Poetry installation guide](https://python-poetry.org/docs/#installation))  
3. **Required Poetry dev dependencies** (installed once):  
   - `pytest`, `pytest-cov`  
   - `coverage`  
   - `cyclonedx-bom` (provides the `cyclonedx-py` CLI)  
   Install with:  
   ```bash
   poetry add --group dev pytest pytest-cov coverage cyclonedx-bom



## Makefile Targets


# Generate a CycloneDX SBOM (JSON + XML) from poetry.lock
sbom:
	poetry run cyclonedx-py poetry -o build/sbom/sbom.json
	poetry run cyclonedx-py poetry -o build/sbom/sbom.xml

# Run tests and generate coverage report
coverage:
	poetry run pytest --cov=. --cov-report=xml:build/coverage/coverage.xml

# Run SonarQube scanner (requires sonar-project.properties)
sonar:
	sonar-scanner \
	  -Dsonar.projectBaseDir=. \
	  -Dsonar.sbom.jsonReportPath=build/sbom/sbom.json \
	  -Dsonar.python.coverage.reportPaths=build/coverage/coverage.xml

# Clean generated artifacts
clean:
	rm -rf build/sbom build/coverage

## Typical SonarQube Properties

Required identifiers sonar.projectKey=my-python-project sonar.projectName=My Python Project sonar.projectVersion=1.0

SonarQube server URL and token must be configured in your local scanner, e.g. via sonar-scanner.properties or environment variables

Source code location sonar.sources=.

Python test coverage report (generated by pytest-cov) sonar.python.coverage.reportPaths=build/coverage/coverage.xml

CycloneDX SBOM (runtime dependencies only) sonar.sbom.jsonReportPath=build/sbom/sbom.json

Encoding sonar.sourceEncoding=UTF-8

## Typical Workflow

Follow these steps to prepare reports and run a full SonarQube analysis.
# 0. (One time) Install dependencies including dev tools
poetry install

# 1. Generate SBOM (runtime dependencies from poetry.lock)
make sbom

# 2. Run tests and generate coverage report
make coverage

# 3. Verify that sonar-project.properties exists in your project root and check all properties
ls sonar-project.properties

# 4. Run SonarQube analysis
make sonar
